## üõ†Ô∏è How to Use

### Opening Diagrams

1. **Online Editor (Recommended)**
   - Go to [app.diagrams.net](https://app.diagrams.net/)
   - Click `File` ‚Üí `Open from` ‚Üí `GitHub`
   - Navigate to this repository and select a `.drawio` file

2. **Desktop Application**
   - Download [Draw.io Desktop](https://github.com/jgraph/drawio-desktop/releases)
   - Clone this repository: `git clone https://github.com/Freaky2112/Draw_IO_Diagrams.git`
   - Open any `.drawio` file with the desktop application

3. **VS Code Extension**
   - Install the [Draw.io Integration extension](https://marketplace.visualstudio.com/items?itemName=hediet.vscode-drawio)
   - Open `.drawio` files directly in VS Code

### Editing Diagrams

1. Open the diagram in Draw.io
2. Make your modifications
3. Save the file (maintains the `.drawio` format)
4. Export as needed (PNG, SVG, PDF, etc.)

### Exporting Diagrams

Draw.io supports multiple export formats:
- **PNG** - For web and presentations (with transparency support)
- **SVG** - For scalable vector graphics
- **PDF** - For documentation
- **JPEG** - For general image use
- **XML** - Raw diagram data

## üìù File Format

All diagrams are saved in `.drawio` format, which is:
- XML-based and human-readable
- Version control friendly
- Compatible with Draw.io web and desktop applications
- Can contain multiple pages/tabs in a single file

## üé® Customization Tips

- **Themes**: Many diagrams use the dark theme (#114B5F background)
- **Colors**: Consistent color schemes for different network zones
- **Icons**: Uses standard networking icons and shapes
- **Labels**: Clear labeling for easy understanding
- **Flow**: Uses arrows and connectors to show data flow

## üí° Best Practices

When creating or modifying diagrams:
- Use consistent color coding for similar components
- Label all connections and interfaces
- Include IP addresses and VLANs where applicable
- Add legends for complex diagrams
- Keep diagrams focused on one concept or system
- Use layers for complex multi-tier architectures
